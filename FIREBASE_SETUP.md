# Firebase Setup Guide

This guide will help you set up Firebase for the Exhibition Tracker application.

## Prerequisites

- A Google account
- Flutter SDK installed
- Android Studio / VS Code with Flutter extensions

## Step 1: Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click "Add project"
3. Enter project name (e.g., "exhibition-tracker")
4. Choose your preferred Google Analytics account or create a new one
5. Click "Create project"
6. Wait for the project to be created, then click "Continue"

## Step 2: Add Android App

1. In the Firebase console, click the Android icon to add an Android app
2. Use the package name from your `android/app/build.gradle` file (default: `com.example.exhibition_app`)
3. Download the `google-services.json` file
4. Place the file in `android/app/google-services.json`
5. Follow the setup instructions in the Firebase console

### Android Configuration

Add the following to your `android/build.gradle`:

```gradle
buildscript {
    dependencies {
        // Add this line
        classpath 'com.google.gms:google-services:4.3.15'
    }
}
```

Add the following to your `android/app/build.gradle`:

```gradle
// Apply the plugin at the bottom of the file
apply plugin: 'com.google.gms.google-services'
```

Add the following to your `android/app/src/main/AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.CAMERA" />
```

## Step 3: Add iOS App (Optional)

1. In the Firebase console, click "Add app" and choose iOS
2. Use the bundle identifier from your `ios/Runner.xcodeproj`
3. Download the `GoogleService-Info.plist` file
4. Place the file in `ios/Runner/GoogleService-Info.plist`
5. Add the Firebase SDK to your `ios/Podfile`:

```ruby
pod 'Firebase/Core'
pod 'Firebase/Auth'
pod 'Firebase/Firestore'
pod 'Firebase/Analytics'
```

6. Run `pod install` in the `ios` directory

## Step 4: Configure Firebase Services

### Enable Authentication

1. In the Firebase console, go to "Authentication" → "Sign-in method"
2. Enable "Google" sign-in:
   - Provide your support email
   - Download the configuration file if needed
3. Enable "Anonymous" sign-in

### Enable Firestore Database

1. In the Firebase console, go to "Firestore Database"
2. Click "Create database"
3. Choose "Start in test mode" (for development)
4. Select a location for your database
5. Click "Create"

### Set up Firestore Security Rules

Replace the default rules with:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can read and write their own data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    // Anyone can read exhibits
    match /exhibits/{exhibitId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
    
    // Users can read/write their own visits
    match /exhibitVisits/{visitId} {
      allow read, write: if request.auth != null && 
        request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && 
        request.auth.uid == request.resource.data.userId;
    }
    
    // Users can read/write their own sessions
    match /userSessions/{sessionId} {
      allow read, write: if request.auth != null && 
        request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && 
        request.auth.uid == request.resource.data.userId;
    }
    
    // Analytics data can be read by authenticated users
    match /exhibitAnalytics/{exhibitId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }
  }
}
```

### Enable Firebase Analytics

1. In the Firebase console, go to "Analytics"
2. Analytics should already be enabled if you set it up during project creation
3. If not, click "Enable Analytics"

## Step 5: Configure FlutterFire

1. Install the FlutterFire CLI:

```bash
dart pub global activate flutterfire_cli
```

2. Run the FlutterFire configuration:

```bash
flutterfire configure
```

This will automatically update your `lib/firebase_options.dart` file with the correct configuration.

## Step 6: Update Firebase Options

After running `flutterfire configure`, your `lib/firebase_options.dart` file should contain the actual configuration values. Replace the placeholder values with the ones generated by the CLI.

## Step 7: Test the Configuration

1. Run the app:

```bash
flutter run
```

2. Try signing in with Google or as an anonymous user
3. Check the Firebase console to see if users are being created
4. Try scanning a QR code and check if data appears in Firestore

## Common Issues and Solutions

### Issue: "google-services.json not found"

**Solution**: Make sure the `google-services.json` file is placed in `android/app/` directory.

### Issue: Camera permission denied

**Solution**: Add camera permissions to your Android manifest and iOS info.plist files.

For Android (`android/app/src/main/AndroidManifest.xml`):

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-feature android:name="android.hardware.camera" android:required="false" />
```

For iOS (`ios/Runner/Info.plist`):

```xml
<key>NSCameraUsageDescription</key>
<string>This app needs camera access to scan QR codes</string>
```

### Issue: Google Sign-In not working

**Solution**: 
1. Make sure you've added the SHA-1 fingerprint to your Firebase project
2. Check that the package name matches exactly
3. Ensure you've set up the OAuth consent screen

### Issue: Firestore permission denied

**Solution**: Check your Firestore security rules and make sure they allow the operations you're trying to perform.

## Production Considerations
